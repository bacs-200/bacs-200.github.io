Chapter 1: Foundations of Web Development

An interactive guide to getting started with GitHub and HTML.

1.1 Introduction to GitHub

Before writing code, you need a place to store, manage, and publish it. We'll use GitHub, the industry standard for version control and code hosting. This section explains the core concepts and guides you through creating your account and the special repository needed for your website.

Version Control (Git)

Think of it as a time machine for your code. It saves "snapshots" (commits) of your project, so you can track every change and revert to previous versions if needed.

GitHub

A website and cloud service that hosts your Git repositories. It's where you'll store your code, collaborate, and showcase your work to the world.

GitHub Pages

A free service from GitHub that takes the HTML, CSS, and JavaScript files in your repository and publishes them as a live website.

The Critical Step: Your Repository Name

To publish a personal website, GitHub requires a repository with a special name. It must be `your-username.github.io`. This is the most common point of error for beginners. Use the tool below to see what your repository name should be.

1.2 Your First Web Page

Now that your repository is set up, it's time to create your first webpage. Every website has a default homepage, which must be named `index.html`. Follow these steps within your new GitHub repository to create and publish this file.

Inside your repository on GitHub, click the `Add file` button and select `Create new file`. In the file name box, type `index.html` exactly.

Copy the code below and paste it into the GitHub text editor.

<!DOCTYPE html>
<html>
<head>
  <title>My Awesome Website</title>
</head>
<body>
  <h1>Hello, World!</h1>
  <p>Welcome to my very first website.</p>
</body>
</html>

Scroll to the bottom of the page. This is called "committing," which means saving your changes. Use a simple message like "Create index.html" and click the green `Commit new file` button.

Go to your repository's `Settings` tab, then click `Pages` in the left menu. Under "Source," select `Deploy from a branch`. Ensure the branch is set to `main`. After saving, GitHub will give you the URL where your new website is live!

1.4 Viewing Source Code vs. Using Developer Tools

One of the best ways to learn is to see how other websites are built. Your browser gives you powerful tools to peek behind the curtain. It's important to understand the difference between simply viewing the source and inspecting the live page.

View Page Source (Ctrl+U)

Shows the raw HTML file exactly as it was sent from the server. It's a static, pure look at the original code. It does not show changes made by JavaScript after the page loads.

Inspect Element (F12 or Right-Click)

This is the developer's primary tool. It shows the *live* page structure (the DOM), including any changes made by JavaScript. You can see CSS styles and even edit the HTML/CSS temporarily to see what happens.
